Hardening fixes from a full review of the runtime code: the panic gesture now always leaves the screen at the restored brightness (an in-flight animation frame could previously land after it), and tapping the widget's Reset while the service is off no longer starts a stuck do-nothing service. Context engine: a rule created or edited moments after an evaluation now applies immediately; the Wi-Fi SSID listener only runs when a Wi-Fi rule actually exists (no more background SSID probing — or root-permission prompts — without one); and a slow SSID read can no longer overwrite a newer connect/disconnect, which could stick a Wi-Fi rule in the wrong state. Super dimming: if Android kills the app while Extra Dim is engaged, the restarted service now clears it instead of leaving the screen dimmed all day. Security review: a corrupted profile import can no longer break the brightness math (NaN values are rejected), and the widget buttons can no longer be triggered by other apps.
